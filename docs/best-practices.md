# Best Practices

Guidelines for running a healthy, growing Discord community.

## Moderation

- **Be consistent.** Apply rules the same way regardless of who breaks them.
- **Document incidents.** Use mod logs and ticket transcripts for appeals.
- **Escalate gradually.** Warn → timeout → kick → ban unless the offense is severe.
- **Train moderators.** New mods should start with limited permissions (Trial Mod).
- **Review AutoMod weekly** during growth spurts to catch false positives.

## Community Culture

- **Lead by example.** Staff should follow the same rules as members.
- **Welcome new members.** A short welcome message or embed sets the tone.
- **Use threads** to keep busy channels readable without splitting into too many channels.
- **Celebrate milestones** (member counts, events) without spamming @everyone.

## Security

- **Enable 2FA** for moderators with ban or manage-server permissions (Server Settings → Safety Setup).
- **Limit Admin count** to people you trust completely.
- **Audit bot permissions** when adding or updating integrations.
- **Never share** mod tokens, bot tokens, or webhook URLs publicly.

## Growth

- **Quality over quantity.** A smaller active server beats a large silent one.
- **Partner carefully.** Only promote servers and creators that fit your community.
- **Collect feedback** through suggestions channels or periodic polls.
- **Update rules and FAQ** when the same questions repeat.

## Server Settings Checklist

| Setting | Recommendation |
|---------|----------------|
| Verification Level | Low or higher for public servers |
| Explicit Media Content Filter | Scan media from all members |
| Default Notifications | Only @mentions |
| System Messages Channel | `#rules` or `#announcements` |
| AFK Channel | Set one in the Voice category |
| Community | Enable for discovery and welcome screen features |

## When Things Go Wrong

| Situation | Action |
|-----------|--------|
| Raid / spam wave | Enable slowmode, pause invites, tighten AutoMod temporarily |
| Mod disagreement | Discuss privately in `#mod-chat`; present united front publicly |
| Rule dispute | Point to written rules; owner makes final call on policy |
| Bot compromise | Kick bot, rotate tokens, audit role changes in audit log |

## Maintenance Schedule

| Frequency | Task |
|-----------|------|
| Weekly | Scan mod logs, review open tickets |
| Monthly | Audit roles, bot list, and channel permissions |
| Quarterly | Update rules, FAQ, and AutoMod wordlists |
